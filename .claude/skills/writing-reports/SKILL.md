# Report Writing

Transform research findings into structured, well-cited reports.

## Purpose

Reports provide:
- Synthesized findings from research
- Clear narrative from complex data
- Proper citations for all claims
- Actionable conclusions

## Report Structure

### 1. Executive Summary (for reports >1000 words)
- 3-5 bullet points of key findings
- Most significant insights
- Main recommendations

### 2. Introduction
- Context and importance
- Research objectives
- Preview of structure

### 3. Key Findings
- Organized by theme or importance
- Clear subheadings
- Citations for all claims [1], [2]

### 4. Analysis
- Connect findings to implications
- Identify patterns and trends
- Bridge to conclusions

### 5. Contradictions/Debates
- Present conflicting viewpoints fairly
- Explain reasons for disagreement
- Avoid taking sides unless evidence is clear

### 6. Conclusion
- Summarize key takeaways
- State implications
- Suggest further research

### 7. References
- Consistent citation format
- All sources mentioned
- Complete and accurate

## Citation Format

Use numbered citations inline:
```markdown
According to recent research [1], this approach is effective.
Multiple studies support this finding [2, 3].

## References
[1] Author, "Title," Source, Date. URL
[2] Author, "Title," Source, Date. URL
```

## Report Types

| Type | Focus | Special Sections |
|------|-------|------------------|
| Technical | Implementation details | Methodology |
| Policy | Recommendations | Action items |
| Comparison | Options analysis | Comparison tables |
| Timeline | Historical context | Chronology |
| Academic | Literature review | Methodology, limitations |

## Best Practices

### Every Claim Needs Support
No unsupported opinions. Cite everything.

### Write for Your Audience
- Technical → precise terminology
- Executive → actionable insights
- General → accessible language

### Use Visual Aids
- Tables for comparisons
- Block quotes for important citations
- Bold for key terms

### Maintain Objectivity
Present evidence, not opinions. If recommending, state criteria.

## Quality Checklist

- [ ] Every claim has citation
- [ ] Logical flow between sections
- [ ] Consistent terminology
- [ ] No unsupported opinions
- [ ] Clear transitions
- [ ] Appropriate length
